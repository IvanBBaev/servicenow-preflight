DEPARTURES · GATES 01–10

The ten checks

Gates 01–02, 05 and 06 board on credentials alone. The rest stay ON HOLD until you supply their inputs — they never depart unchecked. Select a gate for the full read-out.

GATECHECKSTATUS
BOARD KEY

Reading the board

THREE STATUSES
CLEARED The condition holds (pass). Never fails the run.
HOLD Advisory (warn) — not configured, unreachable, or a soft flag. Never fails the run.
DENIED A real problem (fail) that should block the deploy. Fails the run.
0CLEARED

Run completed, no check returned fail.

1BLOCKED

A fail, a zero-match selection, or an unexpected runtime error.

2REJECTED

A usage error — bad flags or config, caught before any check runs.

CONNECTING FLIGHTS

CI, config & the library

.github/workflows/preflight.yml 
name: ServiceNow preflight
on: [workflow_dispatch, pull_request]
jobs:
  preflight:
    runs-on: ubuntu-latest
    permissions: { security-events: write }
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 22 }
      - env:            # repo secrets
          SNPF_INSTANCE: ${{ secrets.SNPF_INSTANCE }}
          SNPF_USER: ${{ secrets.SNPF_USER }}
          SNPF_PASS: ${{ secrets.SNPF_PASS }}
        run: npx servicenow-preflight --format sarif > pf.sarif
      - uses: github/codeql-action/upload-sarif@v3
        if: always()
        with: { sarif_file: pf.sarif }

library api · runPreflight 
import { runPreflight, createSnClient }
  from "servicenow-preflight";

const http = createSnClient({ instanceUrl, auth });
const report = await runPreflight({
  http, scope, updateSetId });
report.ok;      // false if any check failed
report.summary; // { pass, warn, fail }
TYPED ERRORS
Checks call ctx.http, never fetch — so createFakeSnClient unit-tests them with no network.
